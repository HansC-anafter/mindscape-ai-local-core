## 2. Meeting Deliberation

### 2.1 首週節奏原則

- **節奏目標**：先建立角色可信度，再讓觀眾看懂自己，最後用低壓互動留下回訪理由。
- **格式比例**：`4` 支 Reel + `3` 則 Carousel，兼顧觸及、收藏與理解。
- **訊息邏輯**：先講「不是你不夠努力」，再講「你可能還沒切換」，最後講「今晚可以怎麼做」。
- **CTA 邏輯**：前段以 `追蹤`、`收藏` 建立信任，中段用 `留言` 收集痛點，後段用 `分享`、`追蹤` 完成一週閉環。

### 2.2 已整合的必要回修點

- **修正 Day 內容負載**：每一天只保留一個主訊息與一個 CTA，避免一則內容同時教育、轉單、解釋所有問題。
- **修正語氣一致性**：將所有可能過度絕對的句子改成保守、教育式表述。
- **修正受眾對位**：每一天都標明主要受眾層，避免內容節奏忽然偏向太廣或太窄。
- **修正 hooks 對應**：每一支 Reel 都對接 `reel_hook_bank.md` 中的核准 hook，不再使用未審過的臨時開場。

## 5. 每日執行版

### Day 1｜建立第一印象

- **格式**：Reel，`20-30` 秒，單鏡位口播，生活場景拍攝。
- **主要受眾層**：核心受眾。
- **主訊息**：你不是不會休息，而是系統還沒切回放鬆模式。
- **內容框架**：Hook 命中痛點 → 用一句話解釋未切換 → 用一句話交代這週會拆解什麼。
- **CTA**：如果這句話說中你，先追蹤，這週我會用 `7` 天拆開講清楚。
- **對應 Hook**：`Hook 01`。
- **風險註記**：避免把「高警戒」說成診斷；避免用「你一定是」這類絕對語氣。

### Day 2｜建立可收藏的辨識清單

- **格式**：Carousel，`5-6` 頁。
- **主要受眾層**：核心受眾。
- **主訊息**：很多人以為只是累，其實身體正用一些日常訊號提醒你還沒退場。
- **頁面結構**：封面提問 → `3` 個訊號 → 一句提醒 → CTA。
- **CTA**：先收藏，今晚下班後對照看一次。
- **依賴關係**：沿用 D1 語氣與 Day 1 的關鍵詞，確保系列感。
- **風險註記**：訊號描述維持教育用途，不把一般疲勞直接等同臨床狀態。

### Day 3｜拆掉自責式誤解

- **格式**：Reel，`20-25` 秒，節奏略快。
- **主要受眾層**：次核心受眾。
- **主訊息**：睡前停不下來，不一定是你自制力差，也可能是白天沒有完成退場。
- **內容框架**：Hook 拆誤解 → 一句機制翻譯 → 一個今晚可試的小動作。
- **CTA**：留言一個你最常發生的睡前卡住情境，我會優先拆解。
- **對應 Hook**：`Hook 03`。
- **風險註記**：不能把睡前卡住單一歸因，更不能暗示靠一招就能改善睡眠。

### Day 4｜給出最低摩擦微調整

- **格式**：Carousel，`5` 頁，重點句清楚。
- **主要受眾層**：核心受眾。
- **主訊息**：放鬆前先做切換，比硬逼自己休息更有效率。
- **頁面結構**：封面 → 常見錯誤做法 → 切換概念 → `3` 步驟 → CTA。
- **CTA**：先收藏，今晚照這 `3` 步做一次。
- **依賴關係**：與 Day 2 用同一版型語言，降低製作成本與認知負荷。
- **風險註記**：不把 `3` 步驟描述成唯一方法，只說它是起點。

### Day 5｜擴大共鳴與分享動機

- **格式**：Reel，`20-30` 秒，情境剪輯 + 口播。
- **主要受眾層**：核心受眾與外圈受眾。
- **主訊息**：休息不等於恢復；很多時候你不是沒休息，而是還沒真正退場。
- **內容框架**：Hook 命中共鳴 → 對比「躺下」與「退場」 → 一句收斂提醒。
- **CTA**：如果你身邊也有人下班後還像在上班，把這支分享給他。
- **對應 Hook**：`Hook 05`。
- **風險註記**：避免把所有疲累都歸因於未切換，需保留「很多時候」「不一定」等緩衝語。

### Day 6｜讓觀眾自我檢查

- **格式**：Carousel，`4-5` 頁，清單式設計。
- **主要受眾層**：次核心受眾。
- **主訊息**：真正慢下來前，身體通常會先出現幾個小變化。
- **頁面結構**：封面 → `3` 個自我檢查點 → 一句使用方式 → CTA。
- **CTA**：留言告訴我，你最常出現的是哪一個訊號。
- **依賴關係**：可整合 Day 3 留言內容，讓觀眾感到被接住。
- **風險註記**：不把清單講成檢測工具，只當自我觀察提示。

### Day 7｜完成週閉環與下週預告

- **格式**：Reel，`25-35` 秒，回顧式口播。
- **主要受眾層**：全層受眾。
- **主訊息**：週末補眠仍累，未必只是你睡不夠，也可能是整週都沒有真正退場。
- **內容框架**：Hook 回顧痛點 → 串起本週三個核心發現 → 下週預告一個延伸主題。
- **CTA**：如果這週內容有幫你更看懂自己，先追蹤，下週我會繼續拆恢復感。
- **對應 Hook**：`Hook 11`。
- **風險註記**：避免把週末疲累說成睡眠問題結論；需保留教育邊界。

## 6. 固定審稿規則

- 每日內容只能有 `1` 個主結論，不能同時塞進多個教學主軸。
- Reel 前 `3` 秒只做一件事：命中痛點、拆誤解，或提出一個關鍵問題。
- Carousel 每頁只放一個意思，句子盡量短，減少下班後理解成本。
- CTA 不得超出 `追蹤`、`收藏`、`留言`、`分享` 這四類低壓互動。
- 任何提及睡眠、疲累、焦躁的表述，都要避免診斷式結尾。
