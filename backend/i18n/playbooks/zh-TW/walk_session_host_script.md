# 共學場景腳本（簡版 Walk Script）- SOP

## 目標

為素人創作者提供 30 分鐘主持腳本模板，包含：

1. **開場**（5 分鐘）：介紹主題、設定期待、風險提醒
2. **探索**（20 分鐘）：引導參與者觀察、提問、互動
3. **收束**（3 分鐘）：總結本場發現、提出下週任務
4. **回寫引導**（2 分鐘）：引導參與者提供偏好與狀態

**核心價值**：
- 讓創作者能夠順利帶節奏
- 避免主持人無法引導互動
- 確保每場 session 都有結構與產出

## 執行步驟

### Phase 0: 準備工作

**執行順序**：
1. 步驟 0.0: 確認主題與 Lens Card
2. 步驟 0.1: 確認參與者名單
3. 步驟 0.2: 準備主持安全卡

#### 步驟 0.0: 確認主題與 Lens Card

獲取：
- `topic`: 本場主題
- `lens_id`: Lens 識別碼
- `lens_card`: Lens Card（包含判斷標準）

**輸出**：
- `topic`: 主題
- `lens_card`: Lens Card
- `judgment_criteria`: 判斷標準（用於引導）

#### 步驟 0.1: 確認參與者名單

獲取：
- `participants`: 參與者列表
- `participant_count`: 參與者數量

**輸出**：
- `participants`: 參與者列表
- `participant_count`: 參與者數量

#### 步驟 0.2: 準備主持安全卡

從 `creator_cold_start` playbook 獲取或生成：
- `safety_card`: 主持安全卡內容
- `rejection_rules_extension`: 延伸的拒絕規則

**輸出**：
- `safety_card`: 安全卡內容
- `rejection_rules`: 拒絕規則列表

### Phase 1: 開場（5 分鐘）

**時間分配**：
- 0:00-1:00：歡迎與介紹
- 1:00-2:00：主題介紹
- 2:00-3:00：設定期待
- 3:00-4:00：風險提醒（安全卡）
- 4:00-5:00：確認參與者狀態

**腳本模板**：

```
[0:00-1:00] 歡迎與介紹
「大家好，我是 [創作者名稱]。今天我們要一起探索 [主題]。
這不是一個教學，而是一個共同探索的過程。我們一起試試看，看看會發現什麼。」

[1:00-2:00] 主題介紹
「今天的主題是 [主題]。我會分享一些我的觀察，但重點是我們一起探索。
沒有標準答案，只有我們一起發現的東西。」

[2:00-3:00] 設定期待
「今天的目標不是要得到完美的答案，而是要：
1. 一起觀察一些有趣的現象
2. 分享彼此的經驗
3. 發現一些我們之前沒注意到的東西
4. 最後，我們會把今天的發現整理成一些可用的規則」

[3:00-4:00] 風險提醒（安全卡）
「在開始之前，有幾件事要提醒：
1. 我不會提供醫療、法律等專業建議
2. 我不會保證任何效果
3. 這是一個共同探索，不是單向教學
4. 你的選擇和體驗是你自己的責任」

[4:00-5:00] 確認參與者狀態
「在開始之前，我想先了解一下大家的狀態：
- 你今天想要什麼樣的體驗？（安靜/社交/探索/放鬆）
- 你對這個主題有什麼期待？
- 你有什麼想要特別探索的嗎？」
```

**驗收標準**：
- ✅ 完成所有 5 個段落
- ✅ 明確設定期待（不是教學，是共同探索）
- ✅ 完成風險提醒（安全卡內容）
- ✅ 收集參與者狀態

**輸出**：
- `opening_completed`: 開場完成標記
- `participant_states`: 參與者狀態列表

### Phase 2: 探索（20 分鐘）

**時間分配**：
- 5:00-10:00：引導觀察（10 分鐘）
- 10:00-15:00：提問與互動（10 分鐘）
- 15:00-20:00：深入探索（5 分鐘）
- 20:00-25:00：總結觀察（5 分鐘）

**腳本模板**：

```
[5:00-10:00] 引導觀察
「現在我們來一起觀察 [具體場景/現象]。
請大家注意：
1. 你看到了什麼？
2. 你感覺到了什麼？
3. 這讓你想到什麼？

（給參與者 2-3 分鐘觀察，然後分享）

基於我的 Lens，我注意到：
- [判斷句 1]
- [判斷句 2]
- [判斷句 3]

你們有什麼不同的觀察嗎？」

[10:00-15:00] 提問與互動
「我想問大家幾個問題：
1. 這個場景/現象對你來說意味著什麼？
2. 你在什麼狀態下會需要這個？
3. 你覺得什麼人不適合這個？

（引導參與者回答，並記錄關鍵回應）

基於大家的回應，我發現：
- [觀察 1]
- [觀察 2]
- [觀察 3]」

[15:00-20:00] 深入探索
「讓我們再深入一點：
- 如果換一個角度，你會怎麼看？
- 這個發現可以怎麼應用到你的生活中？
- 你覺得還有什麼值得探索的？」

[20:00-25:00] 總結觀察
「讓我們總結一下今天的發現：
1. [發現 1]
2. [發現 2]
3. [發現 3]

這些發現可以怎麼用？我們接下來會整理成一些可用的規則。」
```

**驗收標準**：
- ✅ 完成所有 4 個段落
- ✅ 引導參與者觀察與互動
- ✅ 記錄至少 3 條 lens_notes
- ✅ 收集參與者互動痕跡

**輸出**：
- `lens_notes`: 本場 Lens Notes（至少 3 條）
- `participant_interactions`: 參與者互動記錄
- `key_observations`: 關鍵觀察列表

### Phase 3: 收束（3 分鐘）

**時間分配**：
- 25:00-27:00：總結本場發現
- 27:00-28:00：提出下週任務

**腳本模板**：

```
[25:00-27:00] 總結本場發現
「讓我們總結一下今天的發現：

1. [發現 1] - 這意味著 [意義]
2. [發現 2] - 這意味著 [意義]
3. [發現 3] - 這意味著 [意義]

這些發現可以怎麼用？我們會把它整理成一些可用的規則，寫進你的個人資料包。」

[27:00-28:00] 提出下週任務
「下週我們會繼續探索 [下週主題]。
在下次見面之前，你可以試試：
- [小任務 1]（10-20 分鐘）
- [小任務 2]（可選）

完成後，你可以透過回寫入口告訴我你的發現。我們會把這些發現整理進你的個人資料包。」
```

**驗收標準**：
- ✅ 總結至少 3 個發現
- ✅ 提出至少 1 個下週任務
- ✅ 明確說明會整理進個人資料包

**輸出**：
- `session_summary`: 本場總結
- `next_week_tasks`: 下週任務列表

### Phase 4: 回寫引導（2 分鐘）

**時間分配**：
- 28:00-29:00：引導提供偏好
- 29:00-30:00：引導提供信任證據

**腳本模板**：

```
[28:00-29:00] 引導提供偏好
「在結束之前，我想了解一下你的偏好：
1. 你今天在什麼狀態下參與？（安靜/社交/探索/放鬆）
2. 你對價格的敏感度如何？（低/中/高）
3. 你偏好的氛圍/風格是什麼？（用關鍵詞描述）
4. 有什麼是你絕對不喜歡的？

這些資訊會幫助我更好地理解你，並在未來提供更適合的推薦。」

[29:00-30:00] 引導提供信任證據
「最後一個問題：
在今天這場探索中，哪個瞬間讓你覺得「被理解」或「有共鳴」？

這會幫助我了解什麼樣的內容對你最有價值，我們會把這個記錄進你的信任證據庫。」
```

**驗收標準**：
- ✅ 收集至少 3 個參與者的偏好
- ✅ 收集至少 1 個信任證據
- ✅ 明確說明會記錄進個人資料包

**輸出**：
- `participant_preferences`: 參與者偏好列表
- `trust_evidence`: 信任證據列表

### Phase 5: 記錄 Walk Session

**使用工具**: `walkto_record_walk_session`

**輸入參數**：
- `lens_id`: Lens 識別碼
- `route_map`: 本場路線/節奏（從腳本中提取）
- `lens_notes`: 本場 Lens Notes（至少 3 條）
- `shared_artifacts`: 共享素材（照片、筆記等）
- `duration_minutes`: 30

**驗收標準**：
- ✅ Session 記錄成功
- ✅ 包含至少 3 條 lens_notes
- ✅ 包含參與者互動痕跡
- ✅ 時長 25-35 分鐘

**輸出**：
- `walk_session`: 記錄的 Walk Session
- `session_id`: Session 識別碼

### Phase 6: 生成回寫卡（併入 personal_writeback）

**使用工具**: `walkto_writeback_universe`

**輸入參數**：
- `user_id`: 參與者 ID
- `session_id`: Session ID
- `preferences`: 參與者偏好
- `trust_evidence`: 信任證據
- `state_updates`: 狀態更新

**驗收標準**：
- ✅ 回寫卡包含至少 1 條信任證據
- ✅ 回寫卡包含至少 1 條規則更新
- ✅ 回寫卡包含狀態映射

**輸出**：
- `writeback_cards`: 回寫卡列表
