# 常駐專案同步會議框架

本文檔定義了「Mindscape AI Local Core」專案的常駐同步會議的結構與指南。

## 1. 會議目標

本會議旨在確保所有核心貢獻者對專案的進度、當前風險與下一步計畫有共同的理解。主要目標包括：

-   **同步進度**：快速分享各個模組與功能的開發進度。
-   **識別障礙**：主動提出並討論當前遇到的技術或資源挑戰。
-   **對齊方向**：確保團隊的短期目標與專案的長期藍圖一致。
-   **制定行動**：產生清晰、可執行的下一步行動項目。

## 2. 會議資訊

-   **專案**：Mindscape AI Local Core
-   **頻率**：每週一次，建議安排在 **週一上午 10:00**。
-   **時長**：40 分鐘。
-   **形式**：線上會議。
    -   會議連結：[常設會議室連結] (請替換為實際連結)
    -   會議記錄：重點決議與行動項目將發佈於 Slack `#mindscape-core-dev` 頻道。
-   **參與者**：
    -   **必要人員**：核心貢獻者，特別是 `.github/CODEOWNERS` 中定義的 `@team-leads`。
    -   **可選人員**：根據當週議題邀請相關的功能開發者或合作夥伴。

## 3. 試行議程 (組合式議程)

| 時間 (分鐘) | 議程項目                     | 主持人/負責人 | 說明                                                                       |
| :---------- | :--------------------------- | :------------ | :------------------------------------------------------------------------- |
| 10          | **維運同步 (Operational Sync)**    | SRE/維運代表  | 檢視部署狀態、待處理的資料庫遷移 (`alembic`) 及基礎設施的阻礙。            |
| 20          | **服務整合同步 (Services Sync)** | 各服務代表    | 討論 API 變更、`backend` 與 `gateway` 之間的合約，以及功能開發的依賴關係。 |
| 5           | **架構雷達 (Architectural Radar)**   | 全體          | 提出可能在未來導致技術債務的決策或觀察。                                   |
| 5           | **會議回饋與迭代**           | 全體          | 快速討論本次會議的形式與成效，以便在下次會議進行微調。                     |

## 4. 會前準備

為確保會議高效，請所有參與者在會前：

1.  更新自己的任務狀態。
2.  準備好要分享的進度亮點 (1-2 個)。
3.  思考目前遇到的最大挑戰或需要團隊協助的問題。
